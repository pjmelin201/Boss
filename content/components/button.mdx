## Overview
Buttons must be used consistently and sparingly. The nature of the action performed by the button is supported through variants.

Sometimes, a button is complemented by an icon. Icon only buttons are also common usage.

A counter in the trailing position is a common pattern at Github. Buttons can denote state by changing icon color or text.

Button also acts as a bedrock for the DropdownMenu. Buttons can be grouped in different ways.

Buttons need to visually and non-visually support different user actions and states like hover, disabled, focused and active.

## Buttons with icons

Icons can be used in buttons for different goals:

- A leading icon may be used to amplify the context of the button text
- A leading icon may be used to indicate the button's state
- A trailing icon can be used in buttons as a base for the dropdown menu.

## ButtonLink

The ButtonLink will be a separate component. This will render as an anchor or an appropriate framework link component as configured by the user.

## Accessibility

The Button component has to follow below criteria to be accessible

- Sufficient color contrast during the various states(focus/hover/pressed) of the button in each variant. 
- Correct button text even in icon only buttons
- Make `aria-hidden` for icons.
- Treat link as a link and button as button.

## Button sizes

Button components will be available in two sizes.

Standard(32px) and Small(28px)
